here is what the trade_monitor will do

1. once the sell order market is confirmed, order execution will send the order details to trade monitor module.  details to be sent are:
-TradeID, time(order execution time), strategy (1min or 5min), ticker, shares, entry price(executed price), stop loss(accepted stop market order)

2. trade_monitor module, will insert the details into ActiveTrades table in columns:
tradeID, time(order execution time), strategy, ticker, price, shares, entry_price, stop_loss, date(current date)

3. query table TradeStatus match ticker, strategy and date (current date) set active_trade to open

3.Then trade monitor module will query table TradeParameters and retrieve the LU price matching ticker and the current date and insert the LU price in ActiveTrades column lu_price

LU price is being updated in the TradeParameters table, we need a method to create a trigger that once the LU price is updated in TradeParameters, the lu_price in ActiveTrades is also updated.

4. Trade_monitor will monitor the lu price against the Stop loss price (stop market from order execution) and once the lu price falls below the stop loss price, send an order to replace the stop market price using StopMarket orderID, the new price will be lu current price -$0.05. confirm order was replaced and insert the order in ReplaceStop columns: tradeID, time, ticker, shares, price(replacement price), orderID, status, date.

5.Trade_monitor will monitor the stop market order received from order execution,
If it receives a response form api for status change to executed:
remove the trade details from ActiveTrades table and add to ClosedTrades table columns:
tradeID, strategy, ticker, shares, entry_price, entry_time(order execution time), stop_loss, sl_time(executed stop market time), reason (Stop loss), date, realized((entry price - stop loss price)x shares amount)

6.Trade_monitor will update table TradeStatus matching ticker, strategy and date (current date), add +1 to loss_count and set active_trade to closed

7. if time is 15:55 EST, query table StopMarket, matching tradeID, ticker, shares, date and status=accepted send command CANCEL ALL to cancel all accepted stop market orders.  confirm cancellation. 
send command GET POSITIONS confirm Active trades in ActiveTrades table ticker and shares match open position ticker and shares, if ticker math but share differ, use open position shares, send buy order to buy back open shares for ticker and get confirm executed, inserts order into BuyMarket columns: tradeID, time, ticker, shares, price(executed price), token, orderID, action, status, act_status, notes, date(current date)

8. remove trade from ActiveTrades table and insert details into ClosedTrades table columns:
tradeID, strategy, ticker, shares, entry_price, entry_time(order execution time), exit_price(buy back price), exit_time(executed time), reason (Take Profit), date, realized ((entry price-exit price)xshares amount)

Summary:

-Order_exeuction sends sell market and stop market details to Trade_monitor
-Trade monitor enter trade into ActiveTrades table
-adds LU price to the table
-monitor LU price against Stop market (stop loss) price
-sends replace order if LU falls below Stop market price (stop loss price -$0.05)
-Listen for stop market(stop loss) status update from api connection module
-if stop order = executed, send details to ClosedTrades, remove trade from ActiveTrades table, set TradeStatus table loss_count +1 and active_trade =closed matching ticker, strategy and current date.
-if time is 15:55 est cancel all open stop market open in StopMarket table, close any open trades in ActiveTrades table and move record to ClosedTrades table.
